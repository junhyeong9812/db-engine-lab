# auxPage란? 
앞의 type= {leaf, internal}에 따라서 각각의 역할이 다르다.
type = leaf인 경우
    - 다음 leaf의 page위치를 나타낸다.
type = internal인 경우
    - 처음 자식의 page위치를 나타낸다.
이유는 page가 256엔트리를 넘어가면 split을 통해 페이지를 분리하기 때문에 이렇게 분리되는 구조에서는 해당 내용이 더욱 중요해진다.

# 2.1 non-root leaf split
예시 : MAX_ENTRIES = 4인 leaf L이 꽉 찼고, 여기에 25를 넣어야 된다고 할 때, L에는 이미 다음 형제 s가 존재.
```java
                    ┌───────────────┐
                    │  P  (parent)  │
                    └───────┬───────┘
                            │
                            ▼
              ┌───────────────────────┐        ┌───────────────┐
              │ L: 10  20  30  40     │──aux──▶│ S: 50  60     │
              └───────────────────────┘        └───────────────┘
                        FULL                        (기존 형제)

```
1. Allocate new Right page
    새 페이지를 오른쪽으로 삼는 게 핵심이다. 부모 P는 이미 L을 가리키는 포인터를 갖고 있으니, L을 재자리에 두면 그 포인터가 그대로 유효하다. 부모는 entry 하나만 추가하면 된다. 반대로 새 페이지를 왼쪽에서 만드려면 부모의 기존 포인터까지 고쳐야 하고,
    그게 최 자측 자식(auxPage)이었다면 케이스가 갈라진다. "새것은 항상 오른쪽"이 부모 갱신을 단일 연산으로 묶는 규칙이다.
2. Initialize as LEAF, same parent, auxPage = left.auxPage
   ```java
    
              ┌───────────────────────┐        ┌───────────────┐
              │ L: 10  20  30  40     │──aux──▶│ S: 50  60     │
              └───────────────────────┘        └───────────────┘
                                                       ▲
              ┌───────────────────────┐                │
              │ R: (empty)            │──────aux───────┘
              └───────────────────────┘        S 를 먼저 물려받는다
    ```
   R과 L의 형제니 부모가 같다. 그리고 S를 먼저 R이 인계 받는다. - 4 이게 5번보다 먼저하는 이유가 위와 같다.
3. Move upper half entries to right
   splitPoint = 4/2 = 2 -> 인덱스 2,3(=30,40)이 R로 간다. 큰 키가 오른쪽 - 정렬 불변식이다. 정확히 절반으로 자르니 양쪽 다 여유가 생겨, 다음 insert가 곧바로 또 split을 부르지 않는다.
```java
              ┌───────────┐            ┌───────────┐
              │ L: 10  20 │            │ R: 30  40 │
              └───────────┘            └───────────┘
```

4. left.auxPage = right.pageNumber
    이제 L의 포인터를 R로 덮는다. 2번에서 S를 이미 R이 받았으므로 체인이 이어진다.
```java
   ┌───────────┐        ┌───────────┐        ┌───────────────┐
   │ L: 10  20 │──aux──▶│ R: 30  40 │──aux──▶│ S: 50  60     │
   └───────────┘        └───────────┘        └───────────────┘

   ✗ 순서를 뒤집으면 (4 먼저 → 2):
   ┌───────────┐        ┌───────────┐        ┌───────────────┐
   │ L: 10  20 │──aux──▶│ R: 30  40 │──aux──▶│ R (자기 자신!) │
   └───────────┘        └───────────┘        └───────────────┘
                                              S 는 체인에서 유실
```

5. 새 (key,value)를 key < firstRightKey 기준으로 삽입
   firstRightKey = R.keyAt(0) = 30, 25 < 30 왼쪽,
```java

   ┌───────────────┐        ┌───────────┐
   │ L: 10  20  25 │──aux──▶│ R: 30  40 │
   └───────────────┘        └───────────┘
```
split을 먼저 하고 insert하는 순서가 중요하다.

6. separator = right.keyAt(0)
    R의 최솟값 30이 두 페이지의 경계이다. 그리고 이 키는 R에 그대로 남는다. 
```java

        ┌───────────────┐        ┌───────────┐
        │ L: 10  20  25 │        │ R: 30  40 │
        └───────────────┘        └─────▲─────┘
                                       │
                               separator = 30  (복사 — R 에도 남아 있음)
```
이게 CI-6 ("separator는 right subtree의 leftmost leaf에 존재)
2.3 navigateion의 key == sepKey -> right 규약이 바로 여기서 나온다.

7. insertIntoParent(parent, separator, right.pageNumber)
경계를 부모에게 알린다.
```java
                 ┌─────────────────────────┐
                 │  P:  ... │ 30 │ ...     │
                 └────╱──────────╲─────────┘
                     ╱            ╲
        ┌───────────▼───┐      ┌───▼───────┐        ┌───────────┐
        │ L: 10  20  25 │─aux─▶│ R: 30  40 │──aux──▶│ S: 50  60 │
        └───────────────┘      └───────────┘        └───────────┘
```

# 2.2 root leaf split
leaf가 root일 경우
page 0이 root이자 유일한 leaf고, 꽉 찼을 때, 여기 25를 넣어야 된다면?
```java

   ┌────────────────────────┐
   │ page 0   LEAF  (root)  │
   │ auxPage = -1           │
   │ 10  20  30  40         │   ← 꽉 참
   └────────────────────────┘
```
문제 root는 page 0이여야 한다.
교과서식으로 하면 새 페이지를 만들어 root로 삼는데 이 방법은 루트가 0이라는 조건으로 매칭이 안되고 만약 루트가 3이되면 다시 열었을 때 root를 찾지못하게 된다.
그래서 반대로 한다. root는 page 0에 그대로 두고, 내용물을 아래로 보낸다.

1. Copy root content to new leftChild page
    page 0의 내용을 page 1로 복사한다. 원본은 아직 그대로 둔다.
```java

   ┌────────────────────┐        ┌────────────────────┐
   │ page 0  LEAF       │──복사──▶│ page 1  LEAF       │
   │ 10 20 30 40        │        │ 10 20 30 40        │
   └────────────────────┘        └────────────────────┘
                                    parentPage = 0

```

2. Allocate rightChild as empty LEAF (parent = root)
    page 2를 빈 leaf로 만든다. 부모는 root(0)이다.
    ```
   
   ┌────────────────────┐        ┌────────────────────┐
   │ page 1  LEAF       │        │ page 2  LEAF       │
   │ 10 20 30 40        │        │ (비어 있음)         │
   └────────────────────┘        └────────────────────┘

   ```
3. Move half from leftChild to rightChild
    여기서부터는 2.1과 완전히 똑같다.
    ```
     ┌────────────┐        ┌────────────┐                                                                                                                                                                                           
     │ page 1     │        │ page 2     │                                                                                                                                                                                           
     │ 10  20     │        │ 30  40     │                                                                                                                                                                                           
     └────────────┘        └────────────┘      
   ```

4. Insert new (key, value) into correct side
    page 2의 첫 키가 30이고 25 < 30 -> 왼쪽
    ```
   page 2의 첫 키가 30이고 25 < 30 → 왼쪽.

   ┌────────────────┐    ┌────────────┐
   │ page 1         │    │ page 2     │
   │ 10  20  25     │    │ 30  40     │
   └────────────────┘    └────────────┘
   ```

5. separator = rightChild.keyAt(0)
   page 2의 최솟값 30이 두 페이지의 경계이다.

6. Reinitialize root as INTERNAL
   진짜 사건은 여기다. page 0을 비우고 INTERNAL로 다시 초기화한다.
   rootView.initAsEmpty(Type.INTERNAL, parentPage = INVALID, auxPage = leftChild)                                                                                                                                                    
   rootView.insertAt(0, separator, rightChild.toLong())
    ```
     ┌───────────────────────────────┐                                                                                                                                                                                              
     │ page 0   INTERNAL   ← 타입이 바뀜│                                                                                                                                                                                           
     │ auxPage  = 1                  │                                                                                                                                                                                              
     │ entry[0] = (30, 2)            │                                                                                                                                                                                              
     └───────┬───────────────┬───────┘                                                                                                                                                                                              
             │               │                                                                                                                                                                                                      
             ▼               ▼                                                                                                                                                                                                      
     ┌──────────────┐   ┌──────────────┐                                                                                                                                                                                            
     │ page 1  LEAF │──▶│ page 2  LEAF │                                                                                                                                                                                            
     │ aux = 2      │   │ aux = -1     │                                                                                                                                                                                            
     │ 10  20  25   │   │ 30  40       │                                                                                                                                                                                            
     └──────────────┘   └──────────────┘
   ```
   30은 경계와 짝이 있으니 entry[0] = (30,2)
    page 1은 짝 지을 경계가 없으니 aux = 1
    page 0의 aux를 보면
   1단계 전:  page 0 은 LEAF   →  auxPage = -1   ("다음 형제 없음")
   6단계 후:  page 0 은 INTERNAL →  auxPage = 1    ("최좌측 자식은 1번")


## internal page는 데이터가 아닌 경계선만 가진다. (엔트리 내부)

## internal과 leaf의 각각 entry 구조
```java

   INTERNAL 의 entry[0]              LEAF 의 entry[0]
   ┌─────────┬─────────┐            ┌─────────┬─────────┐
   │  key 8  │  val 8  │            │  key 8  │  val 8  │
   │   30    │    2    │            │   30    │  4000   │
   └─────────┴─────────┘            └─────────┴─────────┘
        ↑         ↑                      ↑         ↑
   separator   자식 페이지 번호        진짜 키    사용자 값
    (경계선)     (포인터)
```

entry는 여전히 (키, 값) 쌍이다. INTERNAL에서 그 키가 separator 역할을 하고, 값은 자식 포인터 뿐이다.

auxPage가 type에 따라 뜻이 바뀐다 entry의 value 절반도 똑같다:

type = LEAF      →  valueAt(i) = 사용자 값
type = INTERNAL  →  valueAt(i) = 자식 페이지 번호
