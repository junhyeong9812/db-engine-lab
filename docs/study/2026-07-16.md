# MutableListOf<Long>
Kotlin에서 비어 있는 변경 가능한 리스트를 만드는 함수
<Long>은 그 안에 담길 원소 타입이 Long(64비트 정수)라는 의미이다.

## 1. mutableListOf vs listOf = "변경 가능" 여부
``` java
val a = listOf(1L, 2L, 3L)          // List<Long> - 읽기 전용
val b = mutableListOf(1L, 2L, 3L)   // MutableList<Long> - 추가/삭제/수정 가능

b.add(4L)
b[0] = 99L
// a.add(4L) 
```
핵심은 타입 레벨의 차이. listOf가 돌려주는 List<T> 인터페이스에는 아예 add,remove, set 같은 메서드가 선언되어 있지 않아, 반면 mutableListOf가 돌려주는 MutableList<T>는 List<T>를 상속하면서 그 변경 메서드들을 추가로 가진다.

2. 내부 구조는 거의 같다.
ListOf와 mutableListOf의 차이는 주로 "겉으로 보여주는 인터페이스"차이이다. 내부 자료구조가 근본적으로 다른게 아니다.

JVM 실 구현
함수 -> 실제 만들어지는 객체
mutableListOf<Long>() -> java.util.ArrayList
listOf(1L, 2L) -> 내부적으로 Arrays.asList 기반의 읽기 전용 래퍼
listOf() -> 싱글톤 EmptyList 객체

mutableListOf()는 내부 구조를 보면 자바의 ArrayList이다.
배열을 들고 있다가 capacity가 다 차면 더 큰배열로 복사해서 늘리는 구조이다.
ListOf로 만든 것도 내부엔 배열이 있지만, add 같은 메서드를 인터페이스에 숨겨서 변경을 막는 것뿐이다.( 완전한 불변(immutable)이 아니라 "읽기 전용 뷰"라는 점이 중요하다.)

3.<Long>에 대한 한가지 함정
long은 객체 타입이라는 리스트에 넣으면 박싱(boxing)이 일어나, 즉 long(원시값 8바이트)가 아니라 java.lang.Long 객체로 ㄱ마싸져서 저장된다.
val list = mutableListOf<Long>() // 원소가 Long 객체(박싱됨)로 저장
val arr = LongArray(10) // 원시 long. 박신 없음, -> 메모리 성능에 유리

성능이 정말 중요한 대량 숫자 저장이라면 LongArray가 훨씬 낫고, 크기가 유동적이고 편의가 필요하다면 mutableListOf를 쓰는 식으로 사용한다.

### 요약
-mutableListOf<Long>() -> 원소를 추가,삭제할 수 있는 빈 리스트, 타입으 MutableList<Long>
- listOf와의 차이는 인터페이스(변경 메서드 노출 여부)가 핵심 -> 내부는 둘 다 배열 기반
- JVM 실체는 ArrayList이고, Long은 박싱되어 객체로 저장됨, 순수 성능이 필요하면 LongArray 고려.

# downTo는 Kotlin에서 숫자를 거꾸로(내림차순) 세는 범위(range)를 만드는 중위 함수이다.
for (k in 1..5)         // 1, 2, 3, 4, 5(올림차순)
for (k in 5 downTo 1)   // 5, 4, 3, 2, 1 (내림차순)
- .. 연산자는 오름차순만 만들 수 있다.
- 그래서 거꾸로 세려면 ..대신 downTo를 사용해야된다.
